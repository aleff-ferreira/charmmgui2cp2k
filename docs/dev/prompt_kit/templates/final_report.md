# Final Report Template

Project: `[PROJECT_NAME]`

Run directory: `[RUN_DIRECTORY]`

Date: `[DATE]`

Access mode: unrestricted full-access mode

## Executive Summary

State the objective, what was completed, the highest-confidence outputs, and the main limitations.

## Objectives

List the project objectives and required deliverables.

## Assumptions And Open Questions

Separate verified facts, provisional assumptions, and unresolved questions.

## Machine Profile And Environment

Summarize hardware, OS, relevant tools, package installs, environment variables, and reproducibility constraints.

## Prior-Art Or Prior-Run Audit

Summarize credible sources, prior outputs, and what was trusted or rejected.

## Input Curation

Describe source inputs, transformations, QC checks, rejected inputs, and provenance.

## Methods And Pipeline Architecture

Describe the workflow, tools, parameters, candidate generation funnel, and validation strategy.

## Validation Results

Report controls, benchmarks, independent checks, statistical/simulation/test evidence, and failure classifications.

## Negative Results

List failed tools, rejected candidates, failed controls, inconclusive tests, and discarded branches.

## Uncertainty And Confidence

Explain confidence levels, uncertainty estimates, sensitivity to assumptions, and residual risks.

## Limitations

State what the work does not prove and what requires human or external validation.

## Ranked Recommendations

Rank final outputs and explain why.

## Reproducibility Index

Link manifests, logs, command histories, environment reports, inputs, outputs, scripts, and validation artifacts.

## Next Actions For Human Review

List concrete next actions before publication, production execution, deployment, or real-world use.
